# 一、v2 核心调整

相比 v1，v2 的核心变化：

| | v1 | v2 |
|---|---|---|
| 需求输入 | 用户自由输入 → LLM 提取偏好 | **预设方案 + 标签选择**，LLM 接口预留 |
| 核心展示 | 课表方案 A/B/C | 课表方案 + **科学选课策略卡片** |
| AI 角色 | 入口依赖 AI | AI 退后一层，做**策略知识增强** |
| Demo 风险 | LLM 提取失败则全流程崩 | **100% 确定性路径**，不依赖 LLM 实时调用 |

**核心原则不变：**

1. 前后端分离
2. 模块尽量简单
3. 优先保证 Demo 闭环
4. 方便后期扩展小程序 / App
5. **确定性路径优先，AI 增强预留**

## 3. Course Engine（核心逻辑层）

### 硬约束 vs 软策略

**硬约束（违反 → 方案直接淘汰）：**

| 硬约束 | 规则 |
|--------|------|
| 时间冲突 | 任意两门课 schedule 不可重叠 |
| Group互斥 | 同一 `credit_transfer_group` 只能选一个班 |
| 专业课必选 | 每个 major group 必须选至少一个班 |
| 学分范围 | 15 ≤ total_credits ≤ 25 |

**软策略（影响评分，不淘汰方案）：**

| 软策略 | 评分形式 |
|--------|---------|
| 不想早八 | 含 1-2节课 → 扣分（`morning_penalty`） |
| 不想周五上课 | 周五有课 → 扣分（`friday_penalty`） |
| 不想周一上课 | 周一有课 → 扣分（`monday_penalty`） |
| 只想上午上课 | 下午/晚间有课 → 扣分（`afternoon_penalty`） |
| 想课程集中 | 碎片时间 → 扣分（降 `compact_score`） |
| 想轻松 | 学分偏高 → 扣分（降 `stress_score`） |
| 想冲GPA | 教师评分偏低 → 扣分（降 `gpa_score`） |

> 关键设计：早八课不会被删掉。如果一位评分5.0的教授只在早八开课，方案仍可能出现——它会因为 `morning_penalty` 扣分，但用极高的 `gpa_score` 补偿。这就是方案 A/B/C 产生差异的来源。

### 职责

- 时间冲突检测
- 课程组合生成
- 多维度评分（含策略惩罚项）

### 课程分类

| course_type | 含义 | 选课行为 |
|-------------|------|---------|
| `"major"` | 专业课 | 每个 `credit_transfer_group` 必须选一个班 |
| `"easy"` | 水课 | 从中选 N 个 group（N 由用户控制，默认 1） |

### 算法工作流

```text
                    用户场景 + easy_count(默认1)
                           │
                           ▼
┌──────────────────────────────────────────┐
│  Step 1: 加载全部课程（不做偏好过滤）       │
│  · 所有课程都参与后续组合                  │
│  · 早八/周五/下午课一律保留                │
└──────────────┬───────────────────────────┘
               ▼
┌──────────────────────────────────────────┐
│  Step 2: 按 course_type 分两组             │
│  · major_groups: 专业课按group聚合        │
│  · easy_groups:  水课按group聚合           │
└──────────────┬───────────────────────────┘
               ▼
┌──────────────────────────────────────────┐
│  Step 3: 穷举专业课组合 + 冲突剪枝         │
│  · 笛卡尔积: 每个major group选1个班       │
│  · 只检查硬约束: 时间冲突 → 跳过          │
│  · 不因早八/周五等偏好淘汰任何组合          │
└──────────────┬───────────────────────────┘
               ▼
┌──────────────────────────────────────────┐
│  Step 4: 水课填充 + 学分范围检查           │
│  · C(K, easy_count) 选N个水课group        │
│  · 每个选中group取评分最高且不冲突的班      │
│  · 总学分 15~25 之外 → 淘汰               │
└──────────────┬───────────────────────────┘
               ▼
┌──────────────────────────────────────────┐
│  Step 5: 评分（含策略惩罚项） → 去重 → Top-3│
│  · 按场景权重 + 惩罚项计算综合得分          │
│  · section_id 集合去重                    │
│  · 返回前3名 (方案A/B/C)                  │
└──────────────────────────────────────────┘
```

### 算法伪代码

```python
def generate_plans(scenario: str, easy_count: int = 1) -> list:
    courses = load_courses()
    weights = SCENARIO_WEIGHTS[scenario]
    
    # ── Step 1: 分组（不过滤，全部课程参与） ──
    major_groups = group_by_transfer_group(
        [c for c in courses if c["course_type"] == "major"]
    )
    easy_groups = group_by_transfer_group(
        [c for c in courses if c["course_type"] == "easy"]
    )
    
    # ── Step 2: 穷举专业课组合（仅冲突检测） ──
    major_combos = []
    for combo in product(*major_groups.values()):
        if not has_conflict(combo):
            major_combos.append(combo)
    
    # ── Step 3: 水课填充 + 学分检查 ──
    plans = []
    easy_group_ids = list(easy_groups.keys())
    for combo in major_combos:
        for picked_ids in combinations(easy_group_ids, 
                                       min(easy_count, len(easy_group_ids))):
            easy_picks = pick_best_no_conflict(combo, picked_ids, easy_groups)
            plan = list(combo) + easy_picks
            
            total_credits = sum(c["credits"] for c in plan)
            if 15 <= total_credits <= 25:      # 硬约束: 学分范围
                plans.append(plan)
    
    # ── Step 4: 评分 → 去重 → Top-3 ──
    scored = []
    for plan in plans:
        total_score, breakdown = calculate_score(plan, weights)
        reasons = generate_reasons(plan, breakdown, scenario)
        analysis = {
            "total_credits": sum(c["credits"] for c in plan),
            "school_days": len(set(s["day"] for c in plan for s in c["schedule"])),
            "earliest_period": min(s["start"] for c in plan for s in c["schedule"]),
            "score_breakdown": breakdown,
            "reasons": reasons
        }
        scored.append((plan, total_score, analysis))
    scored.sort(key=lambda x: x[1], reverse=True)
    unique = deduplicate_with_analysis(scored)
    return unique[:3]


def has_conflict(combo):
    """硬约束: 时间冲突检测"""
    slots = [(s["day"], s["start"], s["end"])
             for c in combo for s in c["schedule"]]
    for i in range(len(slots)):
        for j in range(i+1, len(slots)):
            da, sa, ea = slots[i]
            db, sb, eb = slots[j]
            if da == db and sa < eb and sb < ea:
                return True
    return False


def calculate_score(plan, weights):
    """
    综合得分 = 基础维度分 + 策略惩罚项
    
    基础维度（0~10分）:
      gpa_score:    所有课程 teacher.rating 均值 × 2
      compact_score: 课程时间块连续度
      stress_score:  学分越低越接近满分
      free_score:    完整空闲半天占比 × 10
    
    策略惩罚项（0~10分，不满足时扣分）:
      morning_penalty:   含 1-2节课 → 线性扣分
      friday_penalty:    周五有课 → 扣分
      afternoon_penalty: 下午/晚间有课 → 扣分（仅 morning_only 场景启用）
    """
    
    # ── 基础维度 ──
    ratings = [c["teacher"]["rating"] for c in plan]
    gpa_score = (sum(ratings) / len(ratings)) * 2       # 1-5 → 2-10
    
    compact_score = compute_compactness(plan)             # 0-10
    
    total_credits = sum(c["credits"] for c in plan)
    if total_credits <= 18:
        stress_score = 10.0
    elif total_credits <= 22:
        stress_score = 10.0 - (total_credits - 18) * 1.5
    else:
        stress_score = max(0, 4.0 - (total_credits - 22) * 1.0)
    
    free_score = compute_free_half_days(plan) / 10.0 * 10  # 0-10
    
    # ── 策略惩罚项 ──
    morning_penalty   = compute_morning_penalty(plan)      # 0-10, 早八越多越接近0
    friday_penalty    = compute_friday_penalty(plan)       # 0-10, 周五有课→低分
    monday_penalty    = compute_monday_penalty(plan)       # 0-10, 周一有课→低分
    afternoon_penalty = compute_afternoon_penalty(plan)    # 0-10, 下午有课→低分
    
    # ── 加权求和（直接取key，缺字段会报错，防止隐式溢出）──
    score = (
        gpa_score        * weights["gpa"] +
        compact_score    * weights["compact"] +
        stress_score     * weights["stress"] +
        free_score       * weights["free"] +
        morning_penalty  * weights["morning_penalty"] +
        friday_penalty   * weights["friday_penalty"] +
        monday_penalty   * weights["monday_penalty"] +
        afternoon_penalty * weights["afternoon_penalty"]
    )
    
    breakdown = {
        "gpa_score": round(gpa_score, 1),
        "compact_score": round(compact_score, 1),
        "stress_score": round(stress_score, 1),
        "free_score": round(free_score, 1),
        "morning_penalty": round(morning_penalty, 1),
        "friday_penalty": round(friday_penalty, 1),
        "monday_penalty": round(monday_penalty, 1),
        "afternoon_penalty": round(afternoon_penalty, 1)
    }
    return score, breakdown


def compute_compactness(plan):
    """时间集中度: 课程时间块越连续越高分（0~10）"""
    # 按天聚合所有时间段
    days = {1: [], 2: [], 3: [], 4: [], 5: []}
    for c in plan:
        for s in c["schedule"]:
            days[s["day"]].append((s["start"], s["end"]))
    
    total_fragments = 0
    for slots in days.values():
        if not slots:
            continue
        slots.sort()
        # 计算当天课程块之间的缝隙数量
        for i in range(len(slots) - 1):
            gap = slots[i+1][0] - slots[i][1]
            if gap == 1:
                # 仅隔1节（如4-5之间）= 碎片，严重扣分
                total_fragments += 2
            elif gap == 2:
                # 隔2节（午休）= 可接受，轻微扣分
                total_fragments += 0.5
    
    if total_fragments == 0:
        return 10.0
    elif total_fragments <= 2:
        return 7.0
    elif total_fragments <= 4:
        return 4.0
    else:
        return 1.0


def compute_free_half_days(plan):
    """
    完整空闲半天数: 上午(1-4节)或下午(5-8节)完全无课
    返回值: 空闲半天数 (0~10)
    """
    # 按天聚合
    days = {1: set(), 2: set(), 3: set(), 4: set(), 5: set()}
    for c in plan:
        for s in c["schedule"]:
            days[s["day"]].add(s["start"])
    
    free_count = 0
    for slots in days.values():
        if not slots:
            free_count += 2  # 整天没课 = 两个半天
            continue
        # 上午(1-4节)完全无课
        if all(p >= 5 for p in slots):
            free_count += 1
        # 下午(5-8节)完全无课
        if all(p <= 4 for p in slots):
            free_count += 1
    
    return min(10, free_count)  # 最多10个半天


def compute_morning_penalty(plan):
    """含早八(1-2节) → 扣分。早八课越多扣越多"""
    morning_count = sum(
        1 for c in plan
        for s in c["schedule"]
        if s["start"] <= 2
    )
    if morning_count == 0:
        return 10.0
    elif morning_count <= 2:
        return 7.0
    elif morning_count <= 4:
        return 4.0
    else:
        return 1.0


def compute_friday_penalty(plan):
    """周五有课 → 扣分"""
    has_friday = any(
        s["day"] == 5 for c in plan for s in c["schedule"]
    )
    return 10.0 if not has_friday else 3.0


def compute_monday_penalty(plan):
    """周一有课 → 扣分"""
    has_monday = any(
        s["day"] == 1 for c in plan for s in c["schedule"]
    )
    return 10.0 if not has_monday else 3.0


def compute_afternoon_penalty(plan):
    """下午/晚间有课 → 扣分（morning_only场景用）"""
    has_afternoon = any(
        s["start"] >= 5 for c in plan for s in c["schedule"]
    )
    if has_afternoon:
        count = sum(1 for c in plan for s in c["schedule"] if s["start"] >= 5)
        return max(1.0, 10.0 - count * 1.5)
    return 10.0


def deduplicate_with_analysis(scored):
    """section_id集合去重 (plan, score, analysis)"""
    seen = set()
    unique = []
    for plan, score, analysis in scored:
        key = frozenset(c["section_id"] for c in plan)
        if key not in seen:
            seen.add(key)
            unique.append((plan, score, analysis))
    return unique


def generate_reasons(plan, score_breakdown, scenario):
    """根据评分拆解生成自然语言推荐理由"""
    reasons = []
    
    # 从 score_breakdown 提取高光点
    if score_breakdown.get("morning_penalty", 0) >= 10.0:
        reasons.append("无早八课")
    if score_breakdown.get("friday_penalty", 0) >= 10.0:
        reasons.append("周五全空")
    if score_breakdown.get("monday_penalty", 0) >= 10.0:
        reasons.append("周一全空")
    if score_breakdown.get("afternoon_penalty", 0) >= 10.0:
        reasons.append("下午完全自由")
    
    ratings = [c["teacher"]["rating"] for c in plan]
    avg_rating = sum(ratings) / len(ratings)
    if avg_rating >= 4.5:
        reasons.append(f"教师平均评分{avg_rating:.1f}，整体质量优秀")
    elif avg_rating >= 4.0:
        reasons.append(f"教师平均评分{avg_rating:.1f}")
    
    total_credits = sum(c["credits"] for c in plan)
    if total_credits <= 18:
        reasons.append(f"仅{total_credits}学分，压力很小")
    
    # 策略卡片匹配
    for c in plan:
        if c.get("delivery_mode") == "线上网课" or c.get("delivery_mode") == "线上线下混合":
            reasons.append(f"{c['course_name']}选网课模式，节省通勤时间")
    
    # 如果没有特殊理由，给通用理由
    if not reasons:
        reasons.append("方案综合得分最高")
    
    return reasons
```

### 学分互认 & 平行班示意

```
MATH101 高等数学A
  ├── MATH101-01  王教授  一三五 3-4节  线下  汇文楼301  ⭐评分4.5
  ├── MATH101-02  李教授  二四 1-2节    线下  致理楼201   评分4.2
  └── MATH101-03  网课班  每两周一次    线上  -          评分4.0
         ↑                    ↑
     credit_transfer_group = "MATH101" （选了其中一个，自动排除另外两个）
```

当用户选择预设方案后，Course Engine 在生成课表时：
1. 同一 `credit_transfer_group` 内只保留一个班
2. 优先选评分高的，如果时间冲突则降级到次选
3. 策略卡片可对比同组内不同班的时间成本差异

### Mock数据要求

| 指标 | 要求 |
|------|------|
| 课程总数 | 30~50门 |
| major group | ≥5个（高数、大英、思政、体育、专业必修...），每个group 2-3个平行班 |
| easy group | ≥3个（通识选修、兴趣课...），每个group 1-2个班 |
| 教师数量 | ≥15位，评分有区分度（3.0~5.0） |
| 时间段覆盖 | 第1-2节 ~ 第9-10节，制造"高分课在早八"等冲突场景 |
| 特殊模式 | 至少3门课有"线上/混合"模式可选 |
| location | 汇文楼 + 致理楼，楼层分散 |

**关键**：数据要制造"鱼和熊掌"的张力——高分课在早八、水课和必修冲突、网课名额有限等，这样 Demo 展示时才有故事可讲。
