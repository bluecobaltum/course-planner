# AI 智能选课助手 — 当前阶段架构设计（Hackathon / MVP）

# 一、整体目标

当前阶段目标：

* 两人开发
* 2~3天完成可运行 Demo
* 强调 AI 产品体验
* 保证后续可扩展
* 架构尽量轻量化

因此：

当前架构核心原则：

1. 前后端分离
2. 模块尽量简单
3. 优先保证 Demo 闭环
4. 方便后期扩展小程序 / App

---

# 二、推荐整体架构

```text
Frontend（Web）
    ↓
Backend API（FastAPI）
    ↓
Course Engine（规则 + 评分）
    ↓
LLM Service（AI解释）
    ↓
Database（课程数据）
```

---

# 三、模块划分

# 1. Frontend（前端）

推荐：

* Next.js
* TailwindCSS
* shadcn/ui

负责：

* AI聊天输入
* 课表展示
* 方案切换
* 用户交互
* Demo视觉效果

---

# 2. Backend（核心后端）

推荐：

* Python
* FastAPI

负责：

* API
* 规则处理
* 排课逻辑
* 数据读取
* AI调用

---

# API设计建议

## POST /generate_schedule

输入：

```json
{
  "preferences": {
    "avoid_morning": true,
    "compact_schedule": true,
    "prefer_gpa": true
  }
}
```

输出：

```json
{
  "plans": [
    {
      "score": 8.7,
      "courses": [],
      "analysis": ""
    }
  ]
}
```

# 3. Course Engine（核心逻辑层）

这是整个项目真正核心。

负责：

* 课程筛选
* 冲突检测
* 组合生成
* 评分系统

---

# 当前推荐实现方式

## 第一阶段（Hackathon）

规则系统 + 打分系统。

---

# 工作流

```text
课程数据
    ↓
规则过滤
    ↓
去除冲突
    ↓
生成方案
    ↓
评分排序
```

---

# 示例规则

```python
if avoid_morning:
    remove_8am_courses()
```

```python
if no_friday:
    remove_friday_courses()
```

---

# 评分机制示例

```text
综合得分 =
GPA友好度 * 0.4
时间集中度 * 0.3
压力指数 * 0.2
空闲时间 * 0.1
```

---

# 4. LLM Service（AI解释层）

作用：

不是负责计算。

而是：

* 理解用户自然语言
* 提取偏好
* 生成推荐理由
* 增强 AI 体验

---

# 输入

用户自然语言：

```text
我不想早八，希望轻松一点。
```

---

# 输出

结构化偏好：

```json
{
  "avoid_morning": true,
  "prefer_easy": true
}
```

---

# AI解释示例

```text
由于你更偏向轻松和时间集中，
因此推荐方案A。
```

不是：

“算法多高级”。

# 六、当前阶段不建议做的事情

不要：

* 微服务
* Docker集群
* Redis
* 真推荐系统
* 真机器学习训练
* 教务系统自动登录
* Selenium抢课

会极大拖慢开发速度。

# 八、当前最佳开发策略

当前最推荐：

* AI感

* 交互体验
* Demo完整度

底层算法复杂度。
